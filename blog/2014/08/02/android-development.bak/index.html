Android Development

Aug 2nd, 2014

AndroidManifest.xml

用于描述app的基本特征，并且定义了APP所包含的每个组件(components).
在manifest其中最重要之一的元素就是<uses-sdk>，这个元素声明了app在不通Android版本之间的兼容性，通过android:minSdkVersion和android:targetSdkVersion2个属性
<% codeblock %>
    <manifest xmlns:android="http://schemas.android.com/apk/res/android" ... >
<uses-sdk android:minSdkVersion="8" android:targetSdkVersion="19" />
...
</manifest>
<% end%>

src/

默认包含一个Activity class,当点击app图标启动该APP的时候运行

res/

drawable-hdpi/
layout/
    Directory for files that define your app's user interface.
values/
    Directory for other various XML files that contain a collection of resources, such as string and color definitions.

在真实设备上调试:

在windows平台上开发的话，有可能需要安装合适的USB驱动
在真实设备上允许“USB调试”，此选项在安卓3.2版本或以下在这个路径中设置：Settings>Applications>Development
在安卓4.0或以上版本中，这个路径变为Settings>Develop options
注意：在安卓4.2或以上的版本中，Develop options这个选项默认是隐藏的，需要通过以下方式来显示：Settings>About Phone,然后点击Build Number7次，然后再返回前一个界面就能看到被隐藏的Develop options。
